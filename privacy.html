Politique de confidentialité

Application Undercover Vidéo — Dernière mise à jour : 25 mai 2026

Résumé en une phrase : Undercover Vidéo ne collecte aucune donnée personnelle, ne demande aucun compte, et ne partage rien avec qui que ce soit.

1. Qui est responsable du traitement ?

L'application Undercover Vidéo est éditée par Deponchy Studio (développeur indépendant). Pour toute question concernant cette politique, vous pouvez nous contacter via l'adresse mail liée au compte développeur Google Play.

2. Quelles données sont collectées ?

Aucune donnée personnelle n'est collectée par l'application.

  • Pas de création de compte
  • Pas d'authentification
  • Pas de collecte d'identifiants publicitaires
  • Pas de tracking d'usage, pas d'analytics
  • Pas de géolocalisation
  • Pas d'accès au carnet d'adresses, à la galerie photo, au microphone ou à la caméra

2.1 Données stockées localement sur votre appareil

L'application stocke uniquement, dans la mémoire locale de votre navigateur ou téléphone (localStorage) :

  • La liste des 5 dernières catégories de vidéos utilisées (pour éviter les répétitions).

Ces données ne quittent jamais votre appareil et peuvent être supprimées à tout moment en effaçant les données de l'application.

3. Contenu tiers chargé par l'application

Les vidéos affichées dans l'application sont hébergées sur l'infrastructure GitHub (GitHub Pages). Au premier visionnage, votre appareil télécharge la vidéo depuis ces serveurs ; elle est ensuite mise en cache local pour les visionnages suivants.

GitHub peut, comme tout serveur web, enregistrer techniquement l'adresse IP source des requêtes (logs serveurs standards). Cela ne fait pas l'objet d'une exploitation par Undercover Vidéo.

Politique de confidentialité GitHub : docs.github.com/.../github-general-privacy-statement

3.2 Génération d'images IA (mode Personnalisé Premium)

Le mode Personnalisé (IA), réservé aux utilisateurs Premium, utilise le service tiers Pollinations.ai pour générer des images à partir des idées textuelles que vous saisissez. Lorsque vous lancez une génération :

  • Le texte des 2 idées que vous tapez (ex : « un chat qui danse ») est envoyé à l'API publique image.pollinations.ai.
  • Pollinations reçoit également l'adresse IP source de votre appareil (logs serveurs standards).
  • Aucune autre donnée n'est transmise (pas de nom de joueur, pas d'identifiant).
  • Les images générées sont chargées dans l'application le temps de la partie, puis disparaissent.

N'entrez pas d'informations personnelles ou sensibles dans les idées du mode Personnalisé. Politique de Pollinations : pollinations.ai

4. Permissions Android

L'application ne demande aucune permission sensible. Seul l'accès à internet est utilisé, uniquement pour charger les vidéos depuis le serveur public la première fois.

5. Public et âge minimum

L'application est destinée à tous publics (PEGI 3 / Everyone). Elle ne contient aucun contenu explicite. Les vidéos proviennent de bibliothèques publiques (Pexels, Giphy) filtrées avec une classification adaptée au grand public.

6. Modifications de cette politique

Cette politique peut être mise à jour pour refléter des évolutions de l'application. La date de dernière mise à jour figure en haut de cette page. En cas de changement substantiel, une notification sera affichée dans l'application.

7. Droits utilisateurs (RGPD)

Comme aucune donnée personnelle n'est collectée par l'éditeur, aucune demande RGPD spécifique n'est applicable. Pour effacer les données locales, désinstallez simplement l'application ou videz le cache du navigateur.
